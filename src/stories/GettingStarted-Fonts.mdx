# Fonts

Tint includes HK Grotesk (sans-serif) and Merriweather (serif) fonts in WOFF2
format. These fonts are bundled with the package and can be preloaded.

The fonts are automatically included when you use the `full-bootstrap()` mixin
in your [Styles](/?path=/docs/getting-started-styles--docs) setup.

## Font Preloading

The recommended way to use Tint's fonts is to preload them selectively using the
`getFontPreloads()` helper function. This generates the proper `<link>` tag
props for font preloading.

### Svelte

Use `<svelte:head>` to preload fonts in your Svelte components:

```tsx
<script>
  import { getFontPreloads } from 'tint/fonts'

  // Preload only the fonts you need
  const preloads = getFontPreloads([
    'hkGroteskBold',
    'hkGroteskMedium',
    'merriweatherRegular'
  ])
</script>

<svelte:head>
  {#each preloads as props}
    <link {...props} />
  {/each}
</svelte:head>
```

### Astro

Use the frontmatter section to preload fonts in Astro components:

```astro
---
import { getFontPreloads } from 'tint/fonts'

// Preload specific fonts
const preloads = getFontPreloads(['hkGroteskBold', 'merriweatherRegular'])
---

<head>
  {preloads.map(props => <link {...props} />)}
</head>
```

### Preload All Fonts

If you want to preload all fonts (not recommended unless you're using all of
them):

```javascript
import { getFontPreloads } from 'tint/fonts'

// Omit the array to get all fonts
const preloads = getFontPreloads()
```

## Font Metadata

Each font object includes rich metadata that you can use programmatically:

```javascript
import { fonts } from 'tint/fonts'

// Access individual font metadata
const headingFont = fonts.merriweatherBold

console.log(headingFont)
// {
//   url: "/path/to/Merriweather-Bold.woff2",
//   family: "Merriweather",
//   weight: 700,
//   style: "normal",
//   filename: "Merriweather-Bold"
// }
```

## Best Practices

- **Preload selectively**: Only preload fonts that are used above the fold or
  are critical for initial render
- **Use the helper**: The `getFontPreloads()` function ensures correct
  attributes and type safety
- **Check usage**: Before preloading a font, make sure it's actually used.

## Typography Utilities

Tint provides typography utilities through SASS imports that are included with
the `full-bootstrap()` mixin:

```scss
// Typography utilities and variables
@use 'tint/styles/typography' as typography;

// Font metadata for SASS
@use 'tint/styles/font-face-info' as fonts;
```

### Typography Classes

When you include `full-bootstrap()`, typography utility classes are
automatically generated. You can also generate them separately:

```scss
@use 'tint/styles/bootstrap' as bootstrap;

// Generate only typography classes
@include bootstrap.type-classes();
```

### Apply Typography in SASS

Use the `type-class()` mixin to apply typography styles directly in your SASS:

```scss
@use 'tint/styles/typography' as typography;

.my-heading {
  @include typography.type-class('title-serif-1');
}

.my-body-text {
  @include typography.type-class('body-sans-3');
}
```

## Prose Styles

Tint includes a `.tint--type` class for styling long-form content. This is ideal
for blog posts, articles, or any content-heavy pages:

```html
<article class="tint--type">
  <h1>Article Title</h1>
  <p>
    Paragraphs, lists, code blocks, and other content will be automatically
    styled.
  </p>
  <ul>
    <li>Lists are styled</li>
    <li>With proper spacing</li>
  </ul>
  <pre><code>Code blocks too!</code></pre>
</article>
```

The prose styles are included automatically when you use `full-bootstrap()` or
`base()`.

## Customizing Font Paths

By default, `full-bootstrap()` uses `~tint/fonts` for font paths (your bundler
will resolve this). If you're hosting fonts statically:

```scss
@use 'tint/styles/bootstrap' as bootstrap;

// Custom font path for static hosting
@include bootstrap.full-bootstrap($path: '/fonts');
```

**Note:** Since `full-bootstrap()` already calls `generate-font-faces()`, you
don't need to call it separately unless you want a different path.

### Generate Font Faces Separately

If you need more control, you can generate font-face declarations independently:

```scss
@use 'tint/styles/bootstrap' as bootstrap;

// Generate only font-face declarations with custom path
@include bootstrap.generate-font-faces('/static/fonts');
```

## Typography Imports Reference

### From `tint/styles/bootstrap`:

- `type-classes()` - Generate typography utility classes
- `generate-font-faces($path)` - Generate @font-face declarations
- `full-bootstrap($path)` - Includes everything (recommended)

### From `tint/styles/typography`:

- `type-class($key)` - Apply a specific typography style in SASS (e.g.,
  `title-serif-1`, `body-sans-3`)
